461 /**
462  * @brief Set the edge detection level ratio for the obstacle distance calculation
463  *
464  * This configuration parameter controls the level used for edge detection for a found obstacle.
465  * The value should be in the range from 0 to 1. The obstacle distance is calculated from the
466  * near edge of the peak where the signal amplitude crosses a threshold. This threshold is given
467  * by the ratio parameter times the peak amplitude unless this product is lower than the detection
468  * threshold set by @ref acc_detector_obstacle_configuration_set_thresholds or
469  * @ref acc_detector_obstacle_configuration_set_thresholds. In the latter case,
470  * the detection threshold is used. A zero value of the ratio parameter ignores the peak level and
471  * the reported peak distance tells the position of the near edge of the peak where the amplitude
472  * crosses the peak detection threshold. A value of 1 cuts the edge-detection procedure short and
473  * reports the peak position.
474  *
475  * @param[in] obstacle_configuration The configuration that the setting shall be applied to
476  * @param edge_to_peak_ratio The value to be used for the edge to peak level ratio
477  */
478 void acc_detector_obstacle_configuration_set_edge_to_peak_ratio(acc_detector_obstacle_configuration_t obstacle_configuration,
479  float edge_to_peak_ratio);
