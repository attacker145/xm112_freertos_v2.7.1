7 ## Description
8 
9 The smart presence reference application shows how an application can be woken up by movement and then start to track that movement. It does this by combining usage of Acconeer's Presence Detector in two ways.
10 
11 The first thing that happens is that the reference application configures and creates the Presence Detector for low power consumption and low update rate. Then it activates the detector and starts to detect any type of movement in front of the sensor.
12 
13 ```c
14 if (!execute_wakeup(handle))
15 {
16  /* Handle error */
17 }
18 ```
19 
20 The above function returns when a movement is detected. Now the Presence Detector is configured and created for higher performance and higher update rate. It then starts to track the distance to the movement.
21 
22 ```c
23 if (!execute_movement_tracking(handle))
24 {
25  /* Handle error */
26 }
27 ```
28 
29 When no movement is detected anymore the application goes back to the low power and low update rate execution of the Presence Detector.

31 ## Configuration
32 
33 As mentioned above, the Smart Presence reference application uses the Acconeer Presence Detector, see [Read-the-Docs](https://acconeer-python-exploration.readthedocs.io/en/latest/processing/presence_detection_sparse.html) for a detailed description and acc_detector_presence.h for the API. The Presence Detector is built on top of the Sparse Service, see [Read-the-Docs](https://acconeer-python-exploration.readthedocs.io/en/latest/services/sparse.html) for a detailed description and acc_service_sparse.h for the API.
34
